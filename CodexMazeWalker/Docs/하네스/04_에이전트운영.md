# 에이전트 운영 규약

## 역할 분리

- 메인 에이전트: 범위 확정, 작업 분해, 병렬화 판단, 결과 통합, 최종 의사결정
- generator: 초안 구현, 코드 생산, 자산 경로 초안, 기술 선택 제안
- evaluator: 코드 리뷰, 리스크 점검, 검증 관점 평가, 승인 여부 판단

## 통신 규칙

- generator와 evaluator는 직접 통신하지 않는다.
- 메인 에이전트만 두 에이전트 사이의 결과와 피드백을 중계한다.
- 메인 에이전트는 중계 시 원본 결과와 해석을 분리해서 전달한다.

## generator 입력 패키지

- 현재 목표
- 수정 허용 경로
- 금지 사항
- 완료 조건
- 필요한 경우 이전 evaluator 피드백

## evaluator 입력 패키지

- generator 결과 요약
- 변경 파일 목록
- 의도한 동작
- 실행한 검증 또는 미실행 사유
- 확인이 필요한 리스크 포인트

## 병렬 수행 규칙

- 서로 다른 쓰기 범위를 가진 작업만 병렬 실행한다.
- generator 두 개 이상을 돌릴 때는 파일 소유권을 분리한다.
- evaluator는 직전 배치를 검토하는 동안 다음 generator 배치와 병렬로 운영할 수 있다.
- 토큰 비용보다 리드타임을 우선한다.

## 메인 에이전트 중계 루프

1. 메인 에이전트가 목표와 범위를 문서화한다.
2. generator에 구현 작업을 배정한다.
3. generator 결과를 메인 에이전트가 수집하고 요약한다.
4. 메인 에이전트가 evaluator에 결과와 검증 포인트를 전달한다.
5. evaluator의 판정과 리스크를 메인 에이전트가 다시 정리한다.
6. 메인 에이전트가 수정 재지시 또는 승인 결정을 내린다.
