# 하네스 복원 절차

## 목적

작업 문맥이 오염되었거나, 새 세션에서 처음부터 다시 시작해야 할 때 `Docs/하네스seed` 기준으로 `Docs/하네스`를 재구성한다.

## 절차

1. 현재 `Docs/하네스`를 더 이상 신뢰할 수 없는지 확인한다.
2. 필요하면 기존 작업본을 백업한다.
3. `Docs/하네스seed`의 문서를 `Docs/하네스`로 복사한다.
4. 새 세션의 현재 목표만 `Docs/하네스` 쪽 문서에 다시 적는다.
5. 이전 실행 흔적과 새 목표를 구분한 뒤 작업을 재시작한다.

## 예시 명령

```powershell
$stamp = Get-Date -Format "yyyyMMdd_HHmmss"
if (Test-Path Docs\하네스) {
    Copy-Item Docs\하네스 "Docs\하네스_backup_$stamp" -Recurse
}
New-Item -ItemType Directory -Force Docs\하네스 | Out-Null
Copy-Item Docs\하네스seed\*.md Docs\하네스 -Force
```

## 복원 후 확인

- `Docs/하네스`에 seed와 동일한 파일 세트가 존재하는가
- `00_브리프.md`의 불변 요구사항이 그대로 남아 있는가
- `01_실행계획.md`의 병렬 수행 규칙이 유지되는가
- `04_에이전트운영.md`의 generator/evaluator 분리 규칙이 유지되는가
- `06_시간기록.md`가 비어 있거나, 새 세션 기준으로 다시 기록할 준비가 되어 있는가

## 주의

- 복원은 작업본을 seed 상태로 되돌리는 행위다.
- seed를 수시로 편집하면 복원 기준이 무너지므로 일반 작업 중 수정하지 않는다.
