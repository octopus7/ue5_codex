# CodexPlanet 현재 단계

최종 수정일: 2026-04-04

## 현재 단계 식별자

- 단계 ID: `M1-P0`
- 단계명: 행성 뷰와 코어 런타임 골격 추가

## 선행 정리 상태

- 이전 구현 마일스톤을 이어받는 상태가 아니다.
- 현재 프로젝트는 거의 비어 있는 프로젝트에서 처음부터 작업을 시작하는 상태다.
- `M0`, `M1`은 이번 문서 세트에서 정리한 기준 단계이며, 과거 구현 완료 이력을 뜻하지 않는다.
- 문서 기준으로만 아래 내용이 정리됐다.
  - 이 프로젝트는 탑다운과 무관하다.
  - 핵심 상호작용은 `트랙볼 회전`과 `행성 표면 프랍 배치`다.
  - 행성 본체는 `코드 생성 noisy sphere`, 프랍은 `VOX`라는 방향이 정리됐다.
- 실제로 추가된 행성, 배치, 입력, UI 시스템은 아직 없다.

## 이번 단계 목표

문서로 잠근 컨셉을 바탕으로 실제 구현이 들어갈 최소 런타임 뼈대를 만든다.

이번 단계의 완료 기준은 아래와 같다.
- `GameMode`가 추가된다.
- `PlayerController`가 추가된다.
- 행성 뷰 조작용 `Pawn` 또는 동등한 뷰 제어 액터가 추가된다.
- 행성 자체를 표현하는 `PlanetActor`가 추가된다.
- `PlanetActor`는 코드 생성된 약간의 노이즈 스피어를 초기 행성 표현으로 가진다.
- `BasicMap`에서 기본 진입 시 행성을 중심으로 한 화면 구성이 가능해진다.
- 향후 외부 메시로 교체할 수 있도록 행성 시각 표현과 상호작용 책임이 분리된다.
- 향후 배치 시스템이 `행성 표면`과 `배치된 오브젝트 표면`을 모두 다룰 수 있도록 책임 경계가 준비된다.
- 행성 회전 입력을 올릴 준비가 된 클래스 책임 분리가 정리된다.

## 이번 단계 포함 범위

- 클래스별 책임 분리

- 다음 단계 입력 연결을 위한 최소 준비

- 다음 단계 입력 및 회전 구현 기준

## 이번 단계 전제 조건

- 현재 프로젝트는 여전히 기본 레벨과 기본 모듈만 있는 상태다.
- 실제 구현 기준으로는 지금이 첫 구현 단계다.
- 탑다운 플레이어 이동은 필요하지 않다.
- 플레이어 캐릭터보다 `행성 조작 시점`이 우선이다.
- 행성 본체는 VOX가 아니라 코드 생성 스피어가 우선이다.
- 이후 배치 규칙은 적층 가능, 표면 삼각형 노멀 정렬을 기본값으로 사용한다.
- 이후 행성 `고리 NS`/`위성` 추가 UI는 런타임 조립형이 아니라 에디터에서 확인 가능한 실존 `WBP` 애셋으로 구현한다.

## 구현 메모

- `Character`가 기본 선택이라는 가정을 두지 않는다.
- 행성을 회전시킬지, 카메라를 행성 주위로 회전시킬지는 클래스 책임 관점에서 명확히 나눈다.
- 우선은 `보이는 행성`과 `조작 가능한 화면 구도`를 만드는 데 집중한다.
- 행성 표면 판정과 시각 메시 생성 책임은 가능하면 같은 지점에 강하게 결합하지 않는다.
- 이후 배치 단계에서 행성과 프랍이 같은 배치 표면 개념에 들어올 수 있도록 구조를 열어 둔다.
- 이후 UI 단계에서도 위젯 트리는 런타임 `NativeConstruct` 조립 대신 에디터 코드로 저장된 `WBP` 애셋을 기준으로 잡는다.
- 이 단계에서는 프랍 배치보다 뼈대가 우선이다.
