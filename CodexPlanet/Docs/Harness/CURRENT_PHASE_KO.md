# CodexPlanet 현재 단계

최종 수정일: 2026-04-04

## 현재 단계 식별자

- 단계 ID: `M1-P1`
- 단계명: 프로토타입 방향 잠금과 기본 구조 결정

## 직전 단계 완료 상태

- 직전 단계 `M0-P1` 문서 하네스 작성과 현재 상태 정리는 완료됐다.
- `Docs/Harness` 문서 세트가 생성됐다.
- 현재 프로젝트의 실제 상태가 문서 기준으로 정리됐다.
- 아직 코드, 설정, 애셋 구조는 바뀌지 않았다.

## 이번 단계 목표

빈 프로젝트 상태에서 바로 코드를 늘리기 전에, 첫 프로토타입의 방향을 문서로 잠근다.

이번 단계의 완료 기준은 아래와 같다.
- 프로젝트의 시점이 결정된다.
- 플레이어가 수행할 기본 행동 집합이 결정된다.
- 1차 프로토타입의 핵심 루프가 한 문단 수준으로 명확해진다.
- 첫 구현에 필요한 최소 런타임 클래스 구성이 결정된다.
- 필요한 폴더 구조와 네이밍 규칙의 최소 초안이 정리된다.
- 위 결정들이 `DECISION_LOG_KO.md`에 반영된다.

## 이번 단계 포함 범위

- 시점 결정
- 핵심 루프 결정
- 플레이어 행동 정의
- 최소 런타임 클래스 목록 정리
- 콘텐츠 폴더와 네이밍 초안 정리
- 다음 구현 단계 진입 조건 정리

## 이번 단계 제외 범위

- C++ 클래스 추가
- 입력 애셋 생성
- Blueprint 생성
- UI 제작
- 애셋 제작 자동화
- 맵 수정

## 이번 단계 산출물

- 확정된 프로토타입 방향 메모
- 런타임 클래스 초안 목록
- 폴더/네이밍 규칙 초안
- 갱신된 `DECISION_LOG_KO.md`
- 다음 구현 단계로 이어지는 `CURRENT_PHASE_KO.md` 갱신안

## 이번 단계 전제 조건

- 현재 프로젝트는 `BasicMap`과 기본 런타임 모듈만 가진 초기 상태다.
- 구체 장르나 핵심 루프가 아직 확정되지 않았다.
- 따라서 이 단계에서는 구현보다 결정 정리가 우선이다.

## 구현 메모

- 이 단계에서는 추상적인 희망사항보다 실제 첫 구현에 필요한 항목만 잠근다.
- 확정되지 않은 내용을 억지로 세분화하지 않는다.
- 장르가 확정되기 전에는 전용 시스템 이름, 전용 데이터 애셋 구조, 전용 자동화 규칙을 미리 박지 않는다.
- 다음 단계에서 바로 클래스 추가 작업에 들어갈 수 있을 정도로만 구체화한다.

- 첫 구현에 필요한 런타임 클래스가 정리됐는가
